# Инструкция: базы и обзвоны

## Базы клиентов

База - это сохраненный фильтр клиентов, а не ручной список.

В разделе `Базы клиентов` можно:

- создать базу по сегменту, источнику, категории визита, количеству визитов, давности визита или поиску;
- задать `Срок прозвона` - сколько дней дается на каждого клиента после создания задачи;
- настроить автозадачу: ежедневно или еженедельно, время, исполнитель, режим задачи;
- посмотреть, сколько клиентов подходит сейчас;
- увидеть, сколько клиентов было в последней задаче и насколько база изменилась.

Если база архивирована, она не используется для новых обзвонов. Из архива базу можно восстановить. Удалить навсегда можно только архивную базу, если сервер не видит связанных задач.

## Фиксированная и автообновляемая задача

При создании обзвона из базы есть два режима:

- `Фиксированный список` - клиенты фиксируются на момент создания задачи;
- `Автообновляемая` - CRM синхронизирует состав с текущим фильтром базы.

В автообновляемой задаче новые подходящие клиенты добавляются автоматически. Клиенты, по которым уже была история звонков, не удаляются молча, даже если выпали из фильтра.

## Работа с задачей

В разделе `Задачи обзвона` видно:

- список или канбан задач;
- общий прогресс;
- просрочки;
- контактность;
- конверсию в запись;
- количество попыток.

Внутри задачи по каждому клиенту можно поставить статус:

- `Не звонили`;
- `Не взял трубку`;
- `Перезвонить`;
- `Сомневается`;
- `Записался`;
- `Отказ`.

В результате звонка обязательно пишется саммари. История попыток сохраняется у клиента внутри задачи.

## Массовые действия

В задаче можно выбрать клиентов галочками и применить массовое действие:

- поменять статус;
- поставить общий дедлайн;
- добавить комментарий.

Для каждого выбранного клиента создается запись в истории попыток. Если меняется только комментарий или дедлайн, время фактического контакта не перезаписывается.

## Архив

Задачу можно архивировать, восстановить из архива или удалить навсегда из архива.

Удаление навсегда удаляет саму задачу, ее клиентов и историю попыток. Поэтому оно доступно только из архива и требует подтверждения.

## Как читать отчет

В верхней части задач показывается сводка по текущему фильтру:

- `Задач` - сколько задач в выбранном статусе;
- `Клиентов` - сколько клиентов в этих задачах;
- `Контакт` - доля клиентов, по которым уже был любой результат кроме `Не звонили`;
- `Запись` - доля записавшихся среди тех, с кем был контакт;
- `Просрочено` - сколько клиентов вышли за дедлайн;
- `Попыток` - сколько записей звонков сохранено.

Для автообновляемых задач отчет перед расчетом синхронизирует состав с базой, чтобы цифры не отставали от текущей клиентской базы.
