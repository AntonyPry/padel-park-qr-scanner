# Статус спринтов CRM

Дата фиксации: 2026-05-28.
Последнее обновление: 2026-07-13.

Этот файл - единый источник правды по спринтам. Перед началом нового спринта сначала обновляем этот документ: что уже сделано, что частично, что осталось, какие критерии приемки.

## Статусы

- `done` - задача закрыта как production-функциональность: backend, frontend, права, миграции, UX, проверки.
- `partial` - рабочий слой уже есть, но не закрыты все критерии production-версии.
- `todo` - функциональность еще не реализована.
- `blocked` - нельзя закрыть без внешнего решения или данных.

## Правило работы дальше

Новые задачи не должны создавать новый roadmap поверх старого. Если появляется новая потребность, она добавляется в существующий спринт или оформляется отдельным новым спринтом с явной причиной.

Перед стартом Sprint N:

1. Проверяем этот файл.
2. Отмечаем уже реализованное.
3. Формируем короткий план только по оставшимся пунктам.
4. После разработки обновляем статус и критерии проверки.

## Instructions / Onboarding Roadmap

Цель: держать инструкции CRM живыми, ролевыми и связанными с релизами продукта.

Правило: эти спринты не переименовываем и не переставляем после фиксации. Если появляется новая идея, добавляем ее в ближайший подходящий спринт или создаем новый пункт после Sprint 10.

- strict-режим покрыт unit-тестом;

Осталось закрыть:

- критичных хвостов нет.

Критерии приемки:
