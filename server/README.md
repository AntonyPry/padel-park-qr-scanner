# 🎾 Padel Park Access Control System

Система контроля управления доступом (СКУД) и учета посещаемости для падел-клуба.
Решение объединяет Telegram-бота для регистрации клиентов, физический USB-сканер QR-кодов и веб-панель администратора реального времени.

## ✨ Возможности

### 👤 Для Клиента (Telegram Bot)

- **Регистрация:** Пошаговый сбор данных (Фамилия, Имя, Телефон).
- **Валидация:** Проверка формата имени и номера телефона (+7...).
- **Маркетинг:** Опрос "Откуда вы о нас узнали?".
- **Юридическая защита:** Сбор согласий на обработку ПД и рассылку (галочки перед стартом).
- **QR-пропуск:** Генерация персонального QR-кода (Telegram ID).

### 🖥️ Для Администратора (Web Dashboard)

- **Real-time мониторинг:** Карточки клиентов появляются мгновенно при сканировании (Socket.io).
- **Статус доступа:**
  - 🟢 **Зеленая карточка:** Клиент найден в базе, доступ разрешен.
  - 🔴 **Красная карточка:** Неизвестный QR-код или ошибка.
- **Выдача ключей:** Поле для ввода номера ключа от шкафчика с фиксацией в БД.
- **Надежность:** Сохранение состояния экрана в `localStorage` (данные не пропадают при обновлении страницы).
- **Управление:** Удаление карточек, предотвращение дублей (новая карточка заменяет старую).

### ⚙️ Backend & Hardware

- **Авто-сканер:** Скрипт автоматически находит USB-сканер (по VID/PID) на любом COM-порту.
- **Учет посещений:** Защита от повторных сканирований (интервал 5 минут).
- **База данных:** Mariadb (хранение пользователей и истории визитов).
- **Экспорт:** Выгрузки отчетов и аналитики в Excel (`.xlsx`) через CRM/API.

---

## 🛠 Технологический стек

- **Runtime:** Node.js
- **Bot Framework:** [grammY](https://grammy.dev/)
- **Web Server:** Express + Socket.io
- **Database:** Sequelize (Mariadb dialect)
- **Hardware:** Node SerialPort (чтение данных с USB-сканера в режиме Virtual COM)
- **Process Manager:** PM2 (для автозапуска на Windows)

---

### Автозапуск сервера

```bash
npm install -g pm2 pm2-windows-startup
pm2-startup install
pm2 start bot.js --name padel-bot
pm2 save
```
